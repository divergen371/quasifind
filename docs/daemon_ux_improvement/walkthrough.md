# Walkthrough: Daemon 安定化と UX 改善 (出力フォーマット・色)

デーモンの安定性向上と、CLI の出力機能拡張（フォーマット・色付き出力）を実装しました。

## 修正内容

### 1. デーモンの安定化とセグフォ修正
- **VFS キャッシュのバージョン管理**: $`vfs.ml`$ にマジックナンバーを追加し、構造が変更された (Patricia Trie 導入) 古いキャッシュをロードしてセグフォするのを防ぎました。不整合なキャッシュは自動的に破棄されます。
- **インターン化のスレッドセーフ化**: $`intern.ml`$ の文字列共通化処理を Mutex で保護し、マルチドメイン環境でのメモリ汚染を修正しました。

### 2. デーモンモードのメタデータ同期
- **IPC プロトコルの拡張**: デーモンから CLI へ、ファイルタイプ (`type`)、権限 (`perm`)、正確なサイズ (`Int64`) を送るように拡張しました。
- **表示の乱れ修正**: `Querying daemon...` メッセージの出力を修正し、検索結果と表示が混ざる問題を解消しました。

### 3. UX 改善 (出力フォーマット・色)
- **新モジュール `Formatter`**: `$lib/quasifind/formatter.ml(i)$` を追加。
- **多彩なフォーマット**: `--format` オプションで以下をサポート：
  - `default`: シンプルなパス表示
  - `json`: 詳細な JSON 配列
  - `csv`: 表計算ソフト等で読み込める CSV 形式
  - `table`: 読みやすい固定幅テーブル表示
  - `null`: `xargs -0` 用の null 文字区切り
- **色付き出力**: `--color=always|auto|never` をサポート。ファイルタイプに応じて色付けされます。

## 検証結果

### デーモン安定性
キャッシュの削除・再作成・ロードにおいてセグフォが発生しないことを確認しました。

### 出力フォーマットと色
デーモン経由およびローカル検索の両方で、各フォーマットと色付けが正しく機能することを確認しました。

```bash
# テーブル表示の例
./quasifind . 'name == "main.ml"' --daemon --format=table

# JSON 出力の例
./quasifind . 'name == "dune"' --format=json
```
