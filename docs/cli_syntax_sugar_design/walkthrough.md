# CLI シンタックスシュガー実装の確認 (Walkthrough)

## 概要
人間工学と認知心理学に基づいたCLIフラグ (`-n`, `-e`, `-s` など) を導入し、それらを内部で強力な DSL (Domain Specific Language) へと透過的にコンパイルする機能を `bin/main.ml` に実装しました。

## 修正内容のハイライト
1. **Cmdliner オプションの追加**:
   - `bin/main.ml` に新しく `-n`, `--iname`, `-e`, `-t`, `-s`, `-m`, `-c` の各オプション引数を定義しました。
   - 既存の `search` 関数の引数にそれらを追加し、Cmdliner の `Term.(...)` アプリケーションの中で展開されるよう紐付けました。
2. **動的 DSL 生成器の構築**:
   - `search` 関数の冒頭で各オプションを解釈し、DSL（Ast.Typed.expr）の文字列表現と同等のクエリを動的に組み立てるロジック (`final_expr_str_opt`) を実装しました。
   - ワイルドカード (`*`, `?`) を正規表現へ安全にエスケープする関数を内蔵しました。
3. **意図推論ロジック (Intent Deduction)の導入**:
   - サイズ検索 (`-s`) やコンテンツ検索 (`-c`) のフラグが使われた場合、対象がディレクトリであったことによる予期せぬノイズやエラーを防ぐため、内部で自動的に `type == file` を AND 条件として結合するようにしました。
   - `dune exec quasifind -- . -s "+1KB"` のような入力が `type == file && size > 1KB` として解釈されるようになりました。
4. **ネイティブな短絡評価とゼロアロケーションの維持**:
   - 組み立てられた DSL 文字列は、今まで通りのパーサーに渡されるため、`traversal.ml` で行われる拡張子等のプッシュダウン最適化（Zig レイヤーでの事前間引き）に一切のペナルティを与えない構造になっています。

5. **バグ修正と堅牢性の向上**:
   - **Zig プッシュダウンフィルタの修正**: 拡張子フィルタ (`-e`) などのプッシュダウン最適化がディレクトリまでもフィルタリングしてしまい、再帰的な検索が止まってしまうバグを `lib/quasifind/dirent_stubs.zig` で修正しました。ディレクトリとシンボリックリンクは常に OCaml 側に返し、ファイルのみをフィルタリングするようにしました。
   - **パーサーのバックスラッシュ処理の改善**: `lib/quasifind/parser.ml` において、正規表現内で多用されるバックスラッシュエスケープ（`\.` など）が意図せず削除される問題を解決しました。
   - **正規表現リテラルの採用**: CLI フラグから生成される DSL 内部表現を、ダブルクォート形式から `/.../` 形式に変更し、シェルやパーサーによるエスケープの干渉を受けにくい堅牢な構造に統一しました。

## テストと検証結果
以下のコマンドで動作検証を行い、サブディレクトリ内のファイルも正しく検索されることを確認しました。

```bash
# 拡張子検索 (-e) がサブディレクトリ内 (bench/test.ml など) を正しくヒットさせることを確認
dune exec quasifind -- . -e ml

# 名前検索 (-n) とサイズ検索 (+1KB) の複合
dune exec quasifind -- . -n "*.ml" -s "+1KB"

# コンテンツ検索 (自動的に type == file が付与される)
dune exec quasifind -- . -c "open Quasifind"
```

すべての機能が期待通りに動作し、`quasifind` の高いパフォーマンスを維持しつつ、直感的でミスの少ない CLI インターフェースを実現しました。
